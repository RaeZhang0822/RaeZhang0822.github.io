定义

策略模式：定义一系列算法，把他们一个个封装起来，并且试他们可以相互替换。

使用策略模式重构代码

策略模式指的是定义一系列的算法，把他们一个个封装起来。

将不变的和变化的部分隔开 是每个设计模式的主题，策略模式也不例外，策略模式的目的就 将算法的使用与算法的实现分离开来

一个基于策略模式的程序至少由两个部分组成：
第一部分是一组策略类，策略类封装了具体的算法，负责具体的计算过程。
第二个部分是环境类Context，Context接受客户的请求，随后把请求委托给某一个策略类。

策略模式的优缺点

优点

  1. 利用组合、委托和多态等技术和思想，可以有效地避免多重条件选择语句。
  2. 提供了对开放-封闭原则的完美支持，将算法封装在独立的strategy中,使得它们易于切换，易于理解，易于拓展。
  3. 算法可以复用，从而避免了重复的复制粘贴。
  4. 利用组合和委托让Context拥有执行的能力，也是继承的一种更轻便的替代方案。

缺点

  1. 增加了一些策略类或者策略对象
  2. 必须了解所有的strategy及其各个之间的不同点才能选择一个合适的strategy

总的来说，策略模式的优点远大于缺点，在实际中有大量的应用场景。

一等函数对象与策略模式

在函数作为一等对象的语言中，策略模式是隐形的，strategy也就是值为函数的变量。

在JS中，我们经常用高阶函数来封装不同行为，并且把它传递到另一个函数中，策略模式已经融入到语言本身当中。

小结

在Javascript语言的策略模式中，策略类往往被函数所代替，这时策略模式就成为一种“隐形”的模式。尽管这样，从头到尾地了解策略模式，不仅可以让我们对改模式有更加透彻的了解，也可以使我们明白使用函数的好处。
